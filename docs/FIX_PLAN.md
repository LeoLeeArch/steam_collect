## P0（必须修）

### P0-1 诊断页缺少“最近刷新结果”
- **问题描述**：诊断页只看缓存条目，无法判断最近刷新成功/失败、失败原因。
- **影响范围**：演示现场故障解释能力弱，影响可信度。
- **修复方式**：新增轻量 `RefreshStatusStore`（JSON），`RefreshCoordinator` 在成功/失败后写入摘要；诊断页读取并展示。
- **验证方法**：手动刷新成功/失败后进入诊断页，确认“最近刷新状态/时间/错误信息”更新。

### P0-2 热门榜单缺少历史最低信息直观展示
- **问题描述**：列表只显示当前价/原价/折扣，史低信息需进详情才知道。
- **影响范围**：演示“为什么值得买”不够直观。
- **修复方式**：在热门列表项增加“历史最低”字段，缺失时显示 `--`。
- **验证方法**：热门页可直接看到历史最低字段，不影响点击进入详情。

### P0-3 详情页对“免费候选未接入”提示不够明确
- **问题描述**：用户可能误以为免费候选已实现完整判定。
- **影响范围**：产品预期管理风险。
- **修复方式**：详情页增加“规则状态”说明（免费候选规则未接入，当前重点为价格监控）。
- **验证方法**：进入详情页可见明确提示文案。

### P0-4 设置项对“已实现 vs 预留”边界不够显眼
- **问题描述**：通知开关容易被误解为已经生效。
- **影响范围**：演示中容易被追问“为什么没通知”。
- **修复方式**：在设置页强化说明文案（仅保存配置，不会发通知，不支持自动领取）。
- **验证方法**：设置页可见显式边界提示。

### P0-5 补关键稳定性测试（边界）
- **问题描述**：当前测试覆盖不错，但缺少若干边界测试。
- **影响范围**：收尾阶段缺少“坏数据/边界值”证明。
- **修复方式**：补充测试：缓存损坏回退、near-low 阈值边界、空来源保存失败。
- **验证方法**：`validate_project.sh` + 测试文件可在 Xcode 执行，断言通过。

## P1（建议修）

### P1-1 README 增加“无网络演示话术”
- **问题描述**：演示环境网络不可控时，缺少统一解释模板。
- **影响范围**：演示流畅度。
- **修复方式**：README 增加“无网络时如何演示”小节。
- **验证方法**：README 可直接用于口播说明。

### P1-2 DEMO 脚本补“失败路径演示”
- **问题描述**：现有脚本以成功路径为主。
- **影响范围**：现场异常时临场应对不足。
- **修复方式**：在 `docs/DEMO.md` 加“无网络/源异常时演示策略”。
- **验证方法**：脚本包含 fallback 话术和步骤。

### P1-3 STATUS 更新本轮收尾内容
- **问题描述**：状态文档与本轮验收动作未对齐。
- **影响范围**：交接信息不完整。
- **修复方式**：更新 `docs/STATUS.md` 的完成度、修复项、剩余风险、下一轮建议。
- **验证方法**：状态文档与本轮 commit 一致。

### P1-4 诊断页信息排版收敛
- **问题描述**：文字块可读性一般。
- **影响范围**：非开发观众理解成本。
- **修复方式**：按“刷新状态/缓存状态/说明”分区展示。
- **验证方法**：诊断页层级清晰。

### P1-5 首页空态文案再产品化
- **问题描述**：空态虽有，但可进一步引导“下一步操作”。
- **影响范围**：首次启动体验。
- **修复方式**：空态补“点击刷新”引导语。
- **验证方法**：断网或空缓存时文案明确下一步。
