# UX_NOTES（首次打开用户视角）

日期：2026-03-26
范围：Phase 1 iOS 客户端（首页 / 热门榜单 / 详情页 / 设置页）

## 审查结论（问题与修复）

### 1) 用户第一次打开时，是否知道这个 App 是干什么的？
- **问题**：原首页直接展示列表，缺少一句话价值说明，新用户需要自行猜测用途。
- **修复（低成本）**：在首页新增“首次使用说明”区块，明确 App 目标是追踪热门游戏价格并识别史低/近史低机会。

### 2) 用户是否知道如何手动刷新？
- **问题**：虽有右上角“刷新”按钮，但缺少显式引导，新用户可能不知道要点哪里。
- **修复（低成本）**：在首页“首次使用说明”中明确写出“手动刷新：右上角刷新按钮；首次打开会自动拉取一次”。

### 3) 用户是否知道免费候选和史低的区别？
- **问题**：标签语义未集中解释，用户看见标签但不一定理解判定口径。
- **修复（低成本）**：在首页新增“标签说明”区块，逐条解释：
  - 免费候选：规则待接入（当前仅占位）；
  - 历史最低：当前价等于历史最低价；
  - 接近史低：当前价 ≤ 历史最低价上浮 3%。

### 4) 用户是否知道某个结论是怎么得出的？
- **问题**：详情页显示“接近史低：是/否”，但缺少“为什么是这个结论”的证据文本。
- **修复（低成本）**：详情页新增“判定依据”区块，显示当前价、历史最低价、3%阈值计算后的比较口径。

### 5) 用户是否能顺利从首页进入详情并看到有用信息？
- **现状**：可从首页点进详情，且已有价格字段。
- **增强修复（低成本）**：详情页增加“判定依据”后，信息可用性与可解释性提升，减少“看到了结果但不知道为什么”的困惑。

### 6) 用户是否能在设置页理解每个选项的作用？
- **问题**：设置项虽可操作，但对“Top Sellers/Top Played”和抓取参数影响缺少解释。
- **修复（低成本）**：在设置页“热门来源”“抓取策略”区块补充说明文案，给出默认值建议和性能/完整性权衡提示。

---

## 本次刻意不做（避免高成本）
- 不新增完整引导页（onboarding flow）。
- 不改数据模型与抓取/分析流程。
- 不新增通知实现（仍为预留配置）。

以上策略符合“只修复最明显、最低成本、最高收益”的要求。
